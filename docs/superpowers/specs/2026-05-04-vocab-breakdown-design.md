# Vocab Breakdown — Design Spec

**Date:** 2026-05-04
**Status:** Draft for review
**Owner:** ShadowLearn frontend
**Approach:** B — Structured-first, lazy LLM story, IDB cached

---

## 1. Goal

When a Vietnamese-speaking learner clicks a "breakdown" button on any saved vocab card, open a modal that explains the Chinese word using:

1. **Radical / component decomposition** — visual breakdown of each character.
2. **Sino-Vietnamese (Hán Việt) anchor** — the reading the learner already knows from Vietnamese (`学 = học`, `习 = tập`).
3. **LLM-generated mnemonic story** — a 2–3 sentence Vietnamese story that ties the components together using the Hán Việt readings as memory hooks.

Structural data (radicals, Sino-Viet readings, meanings) renders **instantly from bundled offline lookups**. The mnemonic story is generated by LLM **once** and persisted to IndexedDB — re-opens are instant with no API call.

### Why Vietnamese learners specifically

Vietnamese speakers have a hidden advantage when learning Chinese: a large fraction of Vietnamese vocabulary (~60%) comes from Sino-Vietnamese (Hán Việt) borrowings. Words like 学习 → "học tập", 练习 → "luyện tập", 复习 → "ôn tập" are already in the learner's active Vietnamese vocabulary. Most Chinese learning apps ignore this. Surfacing the Hán Việt reading turns memorization into recognition.

---

## 2. Non-goals (v1)

- User-editable mnemonic stories (could come later as an "Edit story" button).
- Audience languages other than Vietnamese (schema supports adding EN/etc later, UI does not).
- Japanese / non-Chinese vocabulary breakdown.
- Pre-generation on save (rejected — wastes API credits for words the user never inspects).
- Streaming the LLM response (story is short; not worth the complexity).
- Sharing / exporting breakdowns.

---

## 3. User flow

1. User saves a Chinese word to their workbook (existing flow, unchanged).
2. On any vocab card (workbook page, lesson workbook panel), user sees a new **🔍 breakdown** button next to the existing play / remove buttons.
3. User clicks the button → modal opens.
4. **Phase 1 (0 ms)** — modal renders all structural sections immediately from bundled lookups: header, per-character breakdown, Sino-Viet anchor, lesson example. Story section shows a skeleton.
5. **Phase 2 (1–3 s, first open only)** — LLM call fires in the background with all facts pre-injected. Story renders when complete and is persisted to IDB.
6. **Phase 3 (re-open)** — story is in IDB; everything renders instantly with no network call.
7. User closes modal. Cached breakdown persists across sessions.

The same breakdown is reused if the user saves the same word from a different lesson — the IDB store is keyed by the Chinese word, not by the vocab entry ID.

## 4. Architecture

### Data sources

| Source | Role | Where it runs | Size |
|---|---|---|---|
| **Unihan `kVietnamese`** | Sino-Vietnamese reading per character | Frontend bundle (lazy-loaded JSON) | ~70 KB gzipped (~16k chars) |
| **CJKVI IDS / `hanzi-decomp`** | Component decomposition + meanings | Frontend bundle (lazy-loaded) | ~80 KB gzipped |
| **OpenRouter (LLM)** | Generates Vietnamese mnemonic story only | New lightweight backend endpoint `/api/vocab/breakdown-story` | Network |
| **Existing `VocabEntry`** | Word, pinyin, meaning, lesson context | Already in IDB | 0 |

The LLM never produces structural facts — it only writes the creative story given those facts as context. This eliminates hallucinated radicals or wrong Sino-Viet readings.

### Render phases

| Phase | When | What renders | Network |
|---|---|---|---|
| 1 — Instant | Modal opens | Header, per-char breakdown, Sino-Viet anchor, lesson example | None |
| 2 — Lazy | First open only, background | Mnemonic story (animates in) | One LLM call |
| 3 — Cached | Re-open | Everything from IDB | None |
| Error | LLM call fails | Struct sections still visible; story section shows retry | None |

## 5. IndexedDB schema (v10 → v11)

**Migration:** add a new object store `word-breakdowns`. The existing `vocabulary` store is unchanged.

### `word-breakdowns` (new)

| Field | Type | Notes |
|---|---|---|
| `word` | `string` | keyPath. The Chinese word, e.g. `"练习"` |
| `sourceLanguage` | `string` | e.g. `"zh-CN"` — scopes to Chinese for v1 |
| `characters` | `CharData[]` | Per-character structural data (looked up from Unihan + CJKVI) |
| `story` | `string \| null` | LLM-generated Vietnamese mnemonic; `null` until first generation |
| `storyLanguage` | `string` | e.g. `"vi"` — supports future audience languages |
| `generatedAt` | `string \| null` | ISO timestamp of story generation |

### `CharData` shape

```ts
interface CharData {
  char: string            // "练"
  pinyin: string          // "liàn"
  sinoVietnamese: string | null  // "luyện" — null if no Unihan entry
  meaning: string         // "to drill, refine, train"
  components: Component[] // [{ char: "纟", name: "silk thread", meaning: "..." }, ...]
}

interface Component {
  char: string
  name: string            // English/Vietnamese label
  meaning: string         // contextual semantic hint
}
```

### Why a separate store

- A word saved from two different lessons (e.g. user encounters 练习 in lesson A and lesson B) shares one breakdown.
- Keeps `VocabEntry` lean — no schema change needed for that store.
- Cleaner deletion semantics — removing a vocab entry doesn't have to touch the breakdown.

**Why a new endpoint instead of reusing `/api/agent`:** The existing agent router is a streaming UI-message endpoint with tool support (Vercel AI SDK protocol) — overkill for a single one-shot prompt. A dedicated lightweight endpoint is simpler to call from the frontend hook and easier to evolve independently.

## 7. The hook contract

```ts
function useWordBreakdown(word: string, sourceLanguage: string): {
  characters: CharData[]      // always available — looked up sync from local data
  sinoVietnamese: string      // joined Hán Việt reading, e.g. "luyện tập"
  story: string | null        // null while loading, string when ready
  storyLoading: boolean       // true during LLM call
  storyError: Error | null    // populated on failure
  retryStory: () => void      // re-trigger LLM call
}
```

Lifecycle:

1. On mount: synchronously builds `characters` and `sinoVietnamese` from local lookups.
2. Checks IDB `word-breakdowns` for an existing entry.
3. If `story` is present in IDB → returns it immediately.
4. If not → fires the LLM call, persists result to IDB, updates state.
5. On error → exposes error and retry function.

## 8. LLM prompt contract

**System:**

```
You are a Chinese teacher specialising in helping Vietnamese-speaking learners
remember Chinese characters through mnemonic stories.

Write a 2–3 sentence mnemonic story in Vietnamese.

Rules:
- Use the Sino-Vietnamese (Hán Việt) readings provided as memory anchors.
  The learner already knows these from Vietnamese.
- Build a vivid visual scene from the radical/component meanings provided.
- Never invent Sino-Vietnamese readings or component meanings — they are given.
- Keep it concrete, visual, and short.
```

**User:**

```
Word: {word} ({pinyin}) — meaning: {meaning}
Sino-Vietnamese: {sinoVietnamese}

Characters:
{for each char:}
- {char} ({pinyin}) — Hán Việt: {sinoViet}, meaning: {meaning}
  Components: {component1.name} ({component1.meaning}), {component2.name} (...)
{/for}

Write the mnemonic story now.
```

**Model:** same default as existing chat (configurable via OpenRouter key). Single non-streaming call. Expected response < 300 tokens.

**Request / response shape (`POST /api/vocab/breakdown-story`):**

```jsonc
// Request
{
  "word": "练习",
  "pinyin": "liànxí",
  "meaning": "to practice, exercise, drill",
  "sino_vietnamese": "luyện tập",
  "characters": [
    {
      "char": "练",
      "pinyin": "liàn",
      "sino_vietnamese": "luyện",
      "meaning": "to drill, refine",
      "components": [
        { "name": "silk thread", "meaning": "..." },
        { "name": "select / sort", "meaning": "..." }
      ]
    },
    { "char": "习", "...": "..." }
  ],
  "openrouter_api_key": "..."  // optional, falls back to env
}

// Response
{ "story": "Người thợ kéo sợi tơ ... " }
```

## 9. Edge cases

| Case | Behaviour |
|---|---|
| Char missing from Unihan `kVietnamese` | Omit Sino-Viet section for that char; story prompt notes the missing reading. |
| Char missing decomposition (basic chars: 一, 人, 大) | Show char alone without component chips; story uses the character meaning directly. |
| LLM call fails (network, rate limit, etc.) | Struct sections still render; story section shows error message + "Try again" button. |
| No OpenRouter API key configured | Struct sections render; story section shows "Add an OpenRouter key in settings to generate the mnemonic." |
| Source language is not `zh-CN` (Japanese lesson) | Breakdown button hidden — v1 scope is Chinese only. |
| Same word saved from multiple lessons | Single shared breakdown via word-keyed IDB store. |
| User opens modal while offline | Struct sections render normally; story section shows "Offline — story unavailable." |

## 10. Risks & mitigations

| Risk | Mitigation |
|---|---|
| Bundle size creep (~150 KB gzipped for both data files) | Lazy-load both JSON files only when the modal first opens. Modal is gated behind a button click, so the data never ships to users who don't use the feature. |
| LLM story quality variance | Inject rich pre-validated context (radicals + Hán Việt + meanings) so the model can't invent facts. Iterate on the system prompt during implementation. |
| Decomposition data quirks (some chars decompose into rare visual-only components) | Filter component output to a stable allowlist (Kangxi 214 radicals + common semantic components). Unknown components are skipped. |
| User dislikes the generated story | v1 scope omits user editing. If demand emerges, add an "Edit story" affordance that overwrites the cached entry. |

---

## 11. Acceptance criteria

- A 🔍 button appears on every saved Chinese vocab card (workbook page + lesson workbook panel).
- Clicking the button opens a modal that renders structural sections within ~50ms.
- On first open of a new word, the story section shows a skeleton, then the Vietnamese mnemonic appears within 5 seconds (subject to network).
- Re-opening the same word renders the entire modal — including the story — with zero network calls.
- Saving the same word from two different lessons reuses one cached breakdown.
- If the LLM call fails, the user can click "Try again" to retry without re-opening.
- IDB schema upgrades cleanly from v10 to v11 for existing users (no data loss in `vocabulary` store).
- Bundle weight increase: < 200 KB gzipped, lazy-loaded.
